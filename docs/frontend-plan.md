## 2. 路由规划

```
/login                         → 登录页
/                              → 仪表盘（今日概览 + 今日清单PDF导出）
/projects                      → 项目列表（仅显示允许项目）
/projects/:projectId           → 项目详情（船舶列表 + 成员，按项目授权）
/projects/:projectId/ships/:shipId  → 检验项目列表（含批量快速填写模式，按项目授权）
/projects/:projectId/ships/:shipId/observations → 巡检/试航意见列表（按项目授权）
/inspections/:id               → 检验详情（轮次历史 + 结果填写 + 意见管理）
/reports                       → 报表中心
/import                        → 手动导入检验项目（admin/manager，限主管项目）
/admin/users                   → 用户管理（admin）
/admin/projects                → 项目管理（admin/manager，限主管项目）
/admin/ops                     → 数据库运维台（admin）
/admin/import-logs             → 导入日志与异常处理（admin）
/admin/comment-templates       → 意见模板管理
```

```
├──────────────────────────────────────────────────────┤
│  检验项目表格:                                         │
│  ┌────────────────────────────────────────────────┐  │
│  │ # │ 检验项目    │ 专业 │ 计划日期 │ 质检员│ 复检│结果│意见│

│  │ 2 │ 管路系统试压 │ 轮机 │ 04-02  │ 李四 │    │ AA │ 0 │
│  │ 3 │ 船体分段焊接 │ 船体 │ 04-01  │ 王五 │ ✓  │ OWC│ 2 │
```

**交互**：
- 点击行 → 跳转检验详情页
- 筛选器联动，URL 参数持久化（书签/分享可用）
- 结果列显示彩色 Tag（CX 灰 / AA 绿 / QCC 金 / OWC 橙 / RJ 红）
- 意见列显示数字 Badge（开放意见数量，0 不显示）
- 检验员/外部人员只看到允许项目；检验员在允许项目内仍按专业限制可操作，其他专业灰色只读
- 支持 Ant Design Table 的列排序
- **[⚡ 快速填写模式]** 切换按钮，切换后进入批量填写视图

#### 批量快速填写模式

> [!TIP]
> 检验员一天通常检验 10-20 项，逐条打开详情页效率很低。快速填写模式允许在列表页直接批量提交结果。

```
┌──────────────────────────────────────────────────────────┐
│  ⚡ 快速填写模式                          [返回普通模式]   │
├──────────────────────────────────────────────────────────┤
│  筛选: [今日] [我的专业] [待检验]                          │
├──────────────────────────────────────────────────────────┤
│  ☑ │ 检验项目         │ 质检员 │ 结果      │ 备注  │ 意见  │
│  ☑ │ 主机安装对中      │ 张三  │ [AA  ▼]  │ [___] │       │
│  ☑ │ 管路系统试压      │ 李四  │ [QCC ▼]  │ [___] │ [+添加]│
│  ☐ │ 货舱涂装完工      │ 赵六  │ [    ▼]  │ [___] │       │
│  ☑ │ 舵机安装         │ 钱七  │ [OWC ▼]  │ [___] │ [+添加]│
├──────────────────────────────────────────────────────────┤
│  已选 3 项                      [批量提交选中项]           │
└──────────────────────────────────────────────────────────┘
```

**快速填写交互**：
- 默认筛选：今日 + 当前用户专业 + 待检验
- 每行内联编辑：结果下拉框、备注输入框
- 点击 [+添加] 弹出意见 Modal（支持从模板选取）
- 勾选需要提交的行，点击「批量提交」调用 `PUT /api/inspections/batch-result`
- 提交后实时刷新行状态，显示成功/冲突标记

```
│  │                                    [关闭意见]   │  │

├──────────────────────────────────────────────────────┤
│  操作:                                               │
│  [📄 生成 PDF 报告]  [📥 下载 PDF]  [📧 邮件发送报告]     │
└──────────────────────────────────────────────────────┘
```

**交互**：
- 基础信息区域为只读（灰色背景），但 source=manual 的条目管理员/经理可编辑
- 非对应专业的检验员 → 整个页面都是只读
- 轮次 Tab 切换查看历史轮次的结果和意见（历史轮次只读）
- 提交结果时弹出确认对话框
- 乐观锁冲突时 → 弹出提示"数据已被更新，请刷新后重试"
- 添加意见 → 弹出意见 Modal，支持**从模板选取**（下拉搜索 + 微调编辑）
- 关闭意见 → 确认对话框（显示该意见在哪一轮提出）
- 备注字段用于记录非意见类补充信息（如"客户代表在场"、"因天气推迟"）
- "生成 PDF" → 浏览器端 jsPDF 生成，新标签页预览
- "下载 PDF" → 直接下载 PDF 文件
- "邮件发送报告" → 打开 `mailto:` 链接，自动填充项目固定收件人、主题行，提示用户手动附上 PDF

## 6. API 调用层

使用 `axios` + 拦截器统一处理：
- 请求拦截：自动附加 JWT Authorization Header（access_token）
- 响应拦截：
  - **401 + access_token 过期** → 自动调用 `/api/auth/refresh` 换取新 token 后重试原请求（静默刷新）
  - **401 + refresh_token 也过期** → 跳转登录页
  - 409 版本冲突 → 弹出 `VersionConflictModal`
  - 通用错误 → Ant Design message.error Toast
- Base URL 从环境变量读取（开发/生产环境切换）
- Token 存储：localStorage（access_token + refresh_token）
